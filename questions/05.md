# AWS高可用性

## 問題
AWS クラウドにおける高可用性の例は何ですか?

## 選択肢
1. いつでも AWS テクニカル サポートに問い合わせができる
2. リソースに障害が発生した場合でもアプリケーションがアクセス可能であることを保証する
3. オンデマンドで支払うと、あらゆる AWS サービスが利用できるようになる
4. AWS リージョンを使用して世界中のどこにでもデプロイできる

## 正解
2

## 解説
**高可用性（High Availability）** とは、システムが継続的に稼働し続ける能力のことです。

AWS における高可用性の実現方法：

### 1. 複数のアベイラビリティゾーン（AZ）の活用
- 同一リージョン内の複数のAZにリソースを分散配置
- 1つのAZで障害が発生しても、他のAZで継続稼働

### 2. Auto Scaling
- 需要に応じて自動的にインスタンス数を調整
- 障害が発生したインスタンスを自動的に置き換え

### 3. Elastic Load Balancer（ELB）
- 複数のインスタンスに負荷を分散
- 健全性チェックにより、障害インスタンスを自動的に除外

### 4. Amazon RDS Multi-AZ
- データベースを複数のAZに自動レプリケーション
- プライマリDBに障害が発生した場合、自動的にスタンバイに切り替え

### 各選択肢の説明
- **A**: テクニカルサポートは高可用性とは関係ない
- **B**: 正解 - 高可用性の定義そのもの
- **C**: 従量課金制度の説明で、高可用性とは無関係
- **D**: グローバル展開の説明で、高可用性とは異なる概念
