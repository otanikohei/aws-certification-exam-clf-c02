# IAM ルートユーザーのベストプラクティス

## 問題
AWS アカウントのルートユーザーアクセスキーに関する IAM のベストプラクティスは何ですか?

## 選択肢
1. 可能であれば、すべてのルート ユーザー アクセス キーを削除するべき。
2. AWS に保存されている機密情報にアクセスするには、ルートユーザーの認証情報を使用する必要がある。
3. システム管理者グループが日常的なアクセスにルート ユーザー資格情報を使用することを許可する。
4. 実稼働データベース インスタンスにアクセスするには、ルート ユーザーの資格情報を使用する。

## 正解
1

## 解説
**ルートユーザーアクセスキーの削除** は、AWSセキュリティの最重要ベストプラクティスの一つです。

### ルートユーザーの危険性

#### 1. 無制限のアクセス権限
- AWSアカウント内のすべてのリソースへの完全なアクセス権
- IAMポリシーによる制限が適用されない
- 請求情報やアカウント設定の変更も可能

#### 2. 取り消し不可能な権限
- ルートユーザーの権限を制限することは不可能
- 一度漏洩すると甚大な被害の可能性

### IAMベストプラクティス

#### ✅ 推奨される対応
1. **ルートユーザーアクセスキーを削除**
2. **IAMユーザーまたはロールを使用**
3. **MFA（多要素認証）を有効化**
4. **最小権限の原則を適用**

#### ❌ 避けるべき行為
- 日常的な作業でのルートユーザー使用
- ルートユーザー認証情報の共有
- アプリケーションでのルートユーザー使用

### 正しいアクセス管理

```
ルートユーザー
├── アカウント作成時のみ使用
├── MFA設定
├── アクセスキー削除
└── 緊急時のみアクセス

IAMユーザー/ロール
├── 日常的な作業用
├── 必要最小限の権限
├── 定期的な権限見直し
└── アクセスキーローテーション
```

### ルートユーザーが必要な場面
- **アカウント設定の変更**
- **請求情報の管理**
- **サポートプランの変更**
- **アカウントの閉鎖**

### 各選択肢の解説
- **A**: 正解 - セキュリティベストプラクティス
- **B**: 不正解 - IAMユーザー/ロールで十分
- **C**: 不正解 - 日常使用は危険
- **D**: 不正解 - データベースアクセスにルートユーザーは不適切

参考: [AWS IAMベストプラクティス](https://docs.aws.amazon.com/IAM/latest/UserGuide/best-practices.html)
