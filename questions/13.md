# 問題 13

企業にはJava Webアプリケーションがあります。企業は自動デプロイを使用してAWS環境を作成し、アプリケーションの新しいバージョンをデプロイしたいと考えています。これらの要件を満たすAWSサービスはどれですか？

- A) AWS Auto Scaling
- B) AWS Elastic Beanstalk
- C) AWS Control Tower
- D) Amazon EC2

## 正解

**B**

## 解説

### A) AWS Auto Scaling ❌

**不正解**。AWS Auto Scalingは、AWSで実行されるアプリケーションまたはワークロードを監視し、需要に応じてリソース容量を自動的に調整します。AWS Auto Scalingは、アプリケーションをデプロイするために必要なリソースを作成しません。

> 📚 [Auto Scalingについて詳しく学ぶ](https://aws.amazon.com/autoscaling/)

---

### B) AWS Elastic Beanstalk ✅

**正解**。Elastic BeanstalkはWebアプリケーションとサービスをデプロイおよびスケーリングします。まず、コードをElastic Beanstalkにアップロードします。その後、Elastic Beanstalkがデプロイを自動的に管理します。さらに、Elastic Beanstalkはキャパシティプロビジョニング、ロードバランシング、オートスケーリング、アプリケーションヘルスモニタリングを管理します。

> 📚 [Elastic Beanstalkについて詳しく学ぶ](https://aws.amazon.com/elasticbeanstalk/)

---

### C) AWS Control Tower ❌

**不正解**。AWS Control Towerは、AWSのベストプラクティスに従ったマルチアカウント環境を作成および管理する方法を提供します。AWS Control Towerは、アプリケーションをデプロイするために必要なリソースを作成しません。

> 📚 [AWS Control Towerについて詳しく学ぶ](https://aws.amazon.com/controltower/)

---

### D) Amazon EC2 ❌

**不正解**。Amazon EC2は、AWSクラウドでスケーラブルなコンピューティングサーバーを提供します。顧客がEC2インスタンスを管理します。Amazon EC2は、このシナリオで必要な自動デプロイ機能を提供しません。

> 📚 [Amazon EC2について詳しく学ぶ](https://aws.amazon.com/ec2/)
