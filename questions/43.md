# 問題 43

ユーザーは、高速ネットワーク接続が利用できないエリアからAWSにペタバイト規模のデータを移行する必要があります。データを転送する最も速い方法はどれですか？

- A) Amazon S3
- B) AWS Batch
- C) AWS Snowball Edge
- D) AWS VPN

## 正解

**C**

## 解説

### A) Amazon S3 ❌

**不正解**。Amazon S3はオブジェクトストレージサービスですが、高速ネットワーク接続がない場合、ペタバイト規模のデータをインターネット経由でアップロードするのは非常に時間がかかります。

> 📚 [Amazon S3について詳しく学ぶ](https://aws.amazon.com/s3/)

---

### B) AWS Batch ❌

**不正解**。AWS Batchは、バッチコンピューティングジョブを実行するためのサービスです。データ転送サービスではありません。

> 📚 [AWS Batchについて詳しく学ぶ](https://aws.amazon.com/batch/)

---

### C) AWS Snowball Edge ✅

**正解**。Snowball Edgeは、オンボードストレージとコンピューティング能力を備えたハードウェアデバイスです。ローカル環境とAWSクラウド間でデータを転送します。高速ネットワーク接続が利用できない場合、物理デバイスを使用してペタバイト規模のデータを転送するのが最も速い方法です。

> 📚 [Snowball Edgeについて詳しく学ぶ](https://aws.amazon.com/snowball/)

---

### D) AWS VPN ❌

**不正解**。AWS VPNは、オンプレミスネットワークとAWSクラウド間の安全な接続を提供します。ただし、高速ネットワーク接続がない場合、VPN経由でペタバイト規模のデータを転送するのは非常に時間がかかります。

> 📚 [AWS VPNについて詳しく学ぶ](https://aws.amazon.com/vpn/)
