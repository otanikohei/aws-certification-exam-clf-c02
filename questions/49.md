# 問題 49

Amazon EC2インスタンスのCPUとネットワーク使用率を監視するために使用すべきAWSサービスはどれですか？

- A) Amazon Inspector
- B) AWS CloudTrail
- C) Amazon CloudWatch
- D) AWS Config

## 正解

**C**

## 解説

### A) Amazon Inspector ❌

**不正解**。Amazon Inspectorは、ソフトウェアとネットワークのベストプラクティスにおけるセキュリティ脆弱性を検索します。Amazon InspectorはEC2インスタンスをスキャンできますが、CPUとネットワーク使用率を監視しません。

> 📚 [Amazon Inspectorについて詳しく学ぶ](https://aws.amazon.com/inspector/)

---

### B) AWS CloudTrail ❌

**不正解**。CloudTrailは、AWSアカウント内のAPIアクティビティをイベント履歴として記録し、監視、ガバナンス、コンプライアンスに役立ちます。ログは、ユーザーまたはロールによって実行されたアクションなど、AWSアカウントで実行されたアクションを記録します。CloudTrailは、EC2インスタンスのCPUとネットワーク使用率を監視しません。

> 📚 [CloudTrailについて詳しく学ぶ](https://aws.amazon.com/cloudtrail/)

---

### C) Amazon CloudWatch ✅

**正解**。CloudWatchは、リソースを監視し、リアルタイムログを収集します。ログは、リソースの全体的な健全性に関するインサイトを提供できます。EC2インスタンスにCloudWatchエージェントをインストールすると、EC2インスタンスのCPUとネットワーク使用率を監視できます。

> 📚 [CloudWatchについて詳しく学ぶ](https://aws.amazon.com/cloudwatch/)

---

### D) AWS Config ❌

**不正解**。AWS Configは、リソースの過去と現在の設定を分析します。AWS Configは、設定変更を検出して元に戻すことができる修復ルールを使用して、リソース設定をコンプライアンスに準拠させるのに役立ちます。AWS Configは、EC2インスタンスのCPUとネットワーク使用率を監視しません。

> 📚 [AWS Configについて詳しく学ぶ](https://aws.amazon.com/config/)
