# 問題 60

ユーザーは、高速ネットワーク接続が利用できないエリアからAWSにペタバイト規模のデータを移行する必要があります。データを転送する最も速い方法はどれですか？

- A) Amazon S3
- B) AWS Batch
- C) AWS Snowball Edge
- D) AWS VPN

## 正解

**C**

## 解説

### A) Amazon S3 ❌

**不正解**。Amazon S3は、クラウドに大量のデータを保存するための一般的なソリューションです。ただし、S3はデータを移行するために使用できません。

> 📚 [Amazon S3について詳しく学ぶ](https://aws.amazon.com/s3/)

---

### B) AWS Batch ❌

**不正解**。AWS Batchを使用すると、AWSクラウドでバッチコンピューティングワークロードを実行できます。ただし、AWS Batch単体ではデータを移行するために使用できません。

> 📚 [AWS Batchについて詳しく学ぶ](https://aws.amazon.com/batch/)

---

### C) AWS Snowball Edge ✅

**正解**。Snowball Edgeは、データ転送用の堅牢で安全なデバイスを提供します。Snowball Edgeデバイスを使用すると、インターネットを使用せずにデータを物理的に輸送することで、大量のデータを移行できます。これは、インターネット経由でペタバイトの情報を送信しようとするよりも、コスト効率が高く、高速なソリューションです。

> 📚 [Snowball Edgeについて詳しく学ぶ](https://aws.amazon.com/snowball/)

---

### D) AWS VPN ❌

**不正解**。VPNを使用するには、基盤となるネットワークを使用する必要があります。このシナリオでは、この場所では高速ネットワーク接続が利用できません。したがって、VPNはペタバイトのデータを送信する最速のオプションではありません。

> 📚 [AWS Site-to-Site VPNについて詳しく学ぶ](https://aws.amazon.com/vpn/)
